EXTERNAL REST APIs — WEALTH ADVISORY INTELLIGENCE PLATFORM
==========================================================
All external API calls are in: tools/agent_tools.py
No API keys required for any of the 6 data sources.
The only keyed API is Google Gemini (GOOGLE_API_KEY) used by all agents.


──────────────────────────────────────────────────────────────────────────────
1. Yahoo Finance — Nifty 500 (^CRSLDX)
──────────────────────────────────────────────────────────────────────────────
URL       : https://query1.finance.yahoo.com/v8/finance/chart/^CRSLDX?range=3y&interval=1mo
            (formerly ^CNX500, which was delisted on Yahoo Finance)
Function  : fetch_benchmark_returns(risk_tier="MEDIUM")
Phase     : Phase 2 — Risk preference screen + benchmark returns

Why selected:
  The MEDIUM risk tier benchmark is the Nifty 500 Index, which represents the
  broad Indian market (top 500 companies by market cap). Yahoo Finance is the
  most accessible free source for live Indian index historical price data
  without requiring a paid subscription. The 3-year monthly close series is
  fetched to compute the CAGR, giving the RM a live, data-driven benchmark
  return rather than a hardcoded estimate.

Fallback  : Hardcoded 12.5% (historical Nifty 500 long-run average) if call fails.
Auth      : None required. Intermittent HTTP 404/429 due to bot-rate-limiting.


──────────────────────────────────────────────────────────────────────────────
2. Yahoo Finance — Nifty 50 (^NSEI)
──────────────────────────────────────────────────────────────────────────────
URL       : https://query1.finance.yahoo.com/v8/finance/chart/^NSEI?range=3y&interval=1mo
Function  : fetch_benchmark_returns(risk_tier="HIGH")
Phase     : Phase 2 — Risk preference screen + benchmark returns

Why selected:
  The HIGH risk tier benchmark is the Nifty 50 Index, the flagship large-cap
  concentrated Indian equity index. It is the standard benchmark used by
  aggressive equity funds and is the most recognised index for high-risk
  portfolios in India. Same Yahoo Finance source as above for consistency.

Fallback  : Hardcoded 15.0% (historical Nifty 50 long-run average) if call fails.
Auth      : None required. Same rate-limiting behaviour as above.

Note      : NO_RISK (7.1% FD rate) and LOW (7.5% liquid fund) tiers use
            hardcoded RBI-regulated averages and do not call any external API.


──────────────────────────────────────────────────────────────────────────────
3. World Bank Open Data API — India CPI Inflation
──────────────────────────────────────────────────────────────────────────────
URL       : https://api.worldbank.org/v2/country/IND/indicator/FP.CPI.TOTL.ZG
            ?format=json&mrv=5&per_page=5
Function  : fetch_india_inflation_forecast()
Phase     : Phase 3 — Inflation-adjusted (real) returns

Why selected:
  To compute real (inflation-adjusted) portfolio returns, the platform needs
  India's CPI inflation rate. The World Bank Open Data API publishes official
  country-level CPI data (indicator FP.CPI.TOTL.ZG) as a structured, versioned
  REST endpoint — the same data used by economists and regulators globally.
  The last 5 years of data are fetched; the most recent year is used as the
  current CPI and the 5-year average is used as the forecast proxy.

Fallback  : Hardcoded CPI 4.5% (RBI MPC projection) if call fails.
Auth      : None required. Fully public API, generally stable.


──────────────────────────────────────────────────────────────────────────────
4. IMF DataMapper API — India GDP Growth (used in real returns)
──────────────────────────────────────────────────────────────────────────────
URL       : https://www.imf.org/external/datamapper/api/v1/NGDP_RPCH/IND
Function  : fetch_india_inflation_forecast()
Phase     : Phase 3 — Inflation-adjusted (real) returns

Why selected:
  The IMF World Economic Outlook (WEO) is the global standard reference for
  country-level GDP growth projections. The indicator NGDP_RPCH/IND gives
  India's real GDP growth rate, which is used alongside CPI to provide the
  RM with a complete macroeconomic context for the real returns panel
  (portfolio real return vs. GDP growth backdrop).

Fallback  : Hardcoded 7.2% (RBI/IMF consensus estimate) if call fails.
Auth      : None required. Must be called without a browser User-Agent header —
            the IMF CDN returns HTTP 403 for Mozilla/5.0 UA strings. Python's
            default urllib UA (Python-urllib/3.x) is accepted.


──────────────────────────────────────────────────────────────────────────────
5. IMF DataMapper API — India GDP Growth (used in income forecast)
──────────────────────────────────────────────────────────────────────────────
URL       : https://www.imf.org/external/datamapper/api/v1/NGDP_RPCH/IND
Function  : forecast_income_growth()
Phase     : Phase 4 — Income growth forecast (EDD agent)

Why selected:
  The income growth forecast for EDD cases anchors expected career-linked
  income progression to India's nominal GDP growth rate. The logic is that
  in a growing economy, a reasonable professional's income should grow at
  roughly GDP rate, adjusted upward or downward by a career-stage multiplier
  and sector premium. IMF NGDP_RPCH is the most authoritative free source
  for this base rate, making the forecast credible for compliance use.

Fallback  : Hardcoded 7.2% GDP base if call fails.
Auth      : None required. Same endpoint and UA constraint as #4 above.


──────────────────────────────────────────────────────────────────────────────
6. NSE India — EQUITY_L.csv (Listed Companies)
──────────────────────────────────────────────────────────────────────────────
URL       : https://archives.nseindia.com/content/equities/EQUITY_L.csv
Function  : validate_employer_stability()
Phase     : Phase 5 — Employer stability validation

Why selected:
  To assess whether a client's employer is a financially stable, regulated
  entity, the platform checks if the employer is listed on the National Stock
  Exchange (NSE). NSE publishes a complete, daily-updated CSV of all
  NSE-listed companies (EQUITY_L.csv) as a free download. A name match in
  this file confirms the employer is exchange-listed, which is the strongest
  publicly verifiable signal of institutional stability and regulatory
  oversight (SEBI-compliant, audited financials, public disclosures).
  If the employer is not listed, name-pattern heuristics classify the entity
  type (LLP, Pvt Ltd, etc.) for a secondary stability rating.

Fallback  : Name-pattern heuristics (LLP → MEDIUM, unlisted Pvt Ltd → MEDIUM,
            unrecognised → UNKNOWN) if NSE CSV download fails.
Auth      : None required. NSE blocks non-browser User-Agents intermittently;
            the code sends Mozilla/5.0 User-Agent header as a workaround.


──────────────────────────────────────────────────────────────────────────────
7. Google Gemini via Vertex AI (Google ADK) — LLM for all agents
──────────────────────────────────────────────────────────────────────────────
Used by   : All 10 agents (guardrail, client_360, cdd, income_validation, edd,
            portfolio_analysis, loans, expenditure, cibil, risk_assessment,
            report_generation, portfolio_recommendation)
Model     : gemini-2.5-pro on Google Cloud Vertex AI
Config    : GOOGLE_CLOUD_PROJECT + GOOGLE_CLOUD_LOCATION in .env;
            GOOGLE_GENAI_USE_VERTEXAI=1 enables Vertex AI routing
Phase     : Core infrastructure — all phases

Why selected:
  Google ADK (Agent Development Kit) with Gemini is the orchestration
  framework for the entire multi-agent pipeline. Each agent is a Gemini
  LLM instance with tool access to the PostgreSQL databases. Gemini was
  chosen for its function-calling capability, long context window, and
  native ADK integration. Vertex AI was adopted (16 May 2026) for enterprise
  SLA, regional data residency, and production-grade quota management.

Auth      : Google Application Default Credentials (ADC) via gcloud auth.
            No API key required — authenticated via GCP project identity.
            Run `gcloud auth application-default login` before starting.

FALLBACK SUMMARY
==========================================================
All 6 data APIs have hardcoded fallbacks. The pipeline never fails due to
an unreachable external data source. Only Google Gemini/Vertex AI (item 7)
is a hard dependency — if ADC auth fails or quota is exhausted, the pipeline
will not run.

  API                  | Fallback value
  ---------------------|------------------------------------------
  Yahoo Finance MEDIUM | 12.5% CAGR (Nifty 500 historical average)
  Yahoo Finance HIGH   | 15.0% CAGR (Nifty 50 historical average)
  World Bank CPI       | 4.5% (RBI MPC projection)
  IMF GDP (returns)    | 7.2% (RBI/IMF consensus)
  IMF GDP (income)     | 7.2% (RBI/IMF consensus)
  NSE EQUITY_L.csv     | Name-pattern heuristics


==========================================================
IMPLEMENTATION NOTES — BUGS FOUND AND FIXED DURING TESTING
==========================================================
During end-to-end pipeline testing (19 May 2026), all 6 data APIs were found
to be falling back to hardcoded values on every run. Three root causes were
identified and fixed.

──────────────────────────────────────────────────────────────────────────────
Bug 1 — Yahoo Finance: Nifty 500 symbol ^CNX500 delisted
──────────────────────────────────────────────────────────────────────────────
Symptom : fetch_benchmark_returns(risk_tier="MEDIUM") always returned the
          hardcoded fallback of 12.5% instead of a live value.
Root cause: Yahoo Finance delisted the ticker ^CNX500 ("No data found, symbol
          may be delisted"). The HTTP call returned 404, triggering the
          exception handler and the fallback.
Fix     : Changed the MEDIUM tier ticker from ^CNX500 to ^CRSLDX. Yahoo Finance
          confirms ^CRSLDX has longName="NIFTY 500" and currency=INR — it is
          the current active symbol for the same index.
Impact  : MEDIUM-risk client benchmark now returns a live, computed CAGR instead
          of a static estimate. Live value verified: 10.77% (3-year CAGR,
          37 monthly closes as of 19 May 2026).

──────────────────────────────────────────────────────────────────────────────
Bug 2 — IMF DataMapper API: Mozilla/5.0 User-Agent blocked (HTTP 403)
──────────────────────────────────────────────────────────────────────────────
Symptom : Both IMF-dependent functions (fetch_india_inflation_forecast and
          forecast_income_growth) always returned hardcoded fallback GDP of
          7.2%. World Bank CPI was live, but GDP was always static.
Root cause: The IMF DataMapper CDN returns HTTP 403 Forbidden when the request
          carries a browser-style User-Agent header (Mozilla/5.0). The code
          was sending this header to avoid bot-detection, but the IMF
          specifically blocks it. Python's default urllib User-Agent
          (Python-urllib/3.x) is accepted without restriction.
Fix     : Removed the User-Agent header from both IMF urllib.request.Request()
          calls, allowing the default Python UA to be used.
Impact  : IMF GDP now returns live WEO projections. Live value verified:
          6.5% for India (2024 WEO), yielding income_growth_forecast of 6.35%
          for a senior pharma business owner (6.5 × 0.9 multiplier + 0.5%
          sector premium).

──────────────────────────────────────────────────────────────────────────────
Bug 3 — EDD Agent: forecast_income_growth always skipped
──────────────────────────────────────────────────────────────────────────────
Symptom : income_growth_forecast was always null in the final briefing despite
          the EDD agent running and the field being defined in the output schema.
Root cause: forecast_income_growth was Step 4 in the EDD agent's instruction.
          The LLM executed Steps 1–3 (case history, bank statements, RM flags),
          determined that escalation_required = true, and short-circuited its
          output before reaching Step 4 — a well-known LLM behaviour where
          the model terminates early once the decision outcome is clear.
Fix     : Moved forecast_income_growth to Step 1 — the very first tool call,
          before the agent reads any case history or makes any escalation
          decision. The LLM always executes Step 1. Subsequent steps then read
          the case history and render the full finding.
Impact  : income_growth_forecast is now populated on every EDD run. Verified
          on Scenario 2 (CUST000005): projected_growth_rate_pct=6.35,
          consistency_assessment=FLAG_FOR_REVIEW (correct — declared income
          ₹1.5Cr vs spend-inferred ₹3.5Cr despite forecast implying ₹1.7Cr
          trajectory).
