CLAUDE — FIX 2: GUARDRAIL FALLBACK (main.py)
================================================================================

Changed the silent FULL_BRIEFING default to a hard BLOCK on JSON parse
failure.

BEFORE (main.py ~line 343):
  guardrail_result = _extract_json(guardrail_response)
  if not guardrail_result:
      guardrail_result = {
          "guardrail_status": "APPROVED",
          "customer_id":      _extract_customer_id(query),
          "query_intent":     query[:100],
          "approved_for":     "FULL_BRIEFING",
          "notes":            "",
      }

AFTER:
  guardrail_result = _extract_json(guardrail_response)
  if not guardrail_result:
      print("🚫 GUARDRAIL (parse failure): agent returned non-JSON response.")
      return (
          "BLOCKED: The guardrail agent returned an unreadable response. "
          "Please rephrase your query and try again."
      )

Rationale: Any malformed guardrail response previously opened the full
pipeline. The fix ensures a parse failure always blocks, forcing the RM
to rephrase rather than silently running everything.
